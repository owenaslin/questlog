# Tarvn.xyz — Tavern Redesign & Workflow Overhaul

## Context

**Why**: The product (currently "Quest Board" at the soon-to-be `tarvn.xyz` domain) is a working 8-bit gamification productivity tool, but its identity, narrative framing, and core loop (collect → track → complete → improve profile) feel disjointed. The 8-bit visuals exist, but the *world* doesn't — there's no sense of place, the AI quest generator is isolated on its own page, the profile is a long single-page dump, and there's nothing social or shareable to make the "improve profile" step rewarding outside of personal use.

**Outcome**: Reframe the entire app as a **virtual tavern** — the place adventurers gather to pick up quests, track their journey, celebrate completions, and show off their hero. The existing 8-bit aesthetic is preserved and *extended* with warm tavern accents (candlelight gold, ember red, oak brown). Workflows are restructured around four "rooms" of the tavern: **The Quest Board** (collect), **My Journal** (track), **The Hearth** (complete & celebrate), and **The Hero** (profile + share).

---

## Codebase Snapshot (what already exists)

- **Stack**: Next.js 15 App Router, React 18, Tailwind 3, Supabase (auth + Postgres + RLS), Google Gemini for AI quest gen.
- **Visuals**: "Press Start 2P" font, 16-color retro palette (`tailwind.config.ts`), CRT scanlines + pixel borders + step-based animations in [globals.css](quest-board/src/app/globals.css).
- **Data**: `profiles` (xp_total, level), `quests` (type, source, difficulty, status), `user_quests` (per-user state), `badges` + `user_badges`, `questlines` + `questline_steps` + `user_questline_progress`, `streaks`. All RLS-protected.
- **Pages**: [/](quest-board/src/app/page.tsx), [/quests](quest-board/src/app/quests/page.tsx), [/quests/[id]](quest-board/src/app/quests/[id]), [/quests/new](quest-board/src/app/quests/new), [/generate](quest-board/src/app/generate), [/profile](quest-board/src/app/profile/page.tsx), [/questlines](quest-board/src/app/questlines), [/categories](quest-board/src/app/categories), [/badges](quest-board/src/app/badges), auth pages.
- **Components**: 15 components incl. `PixelButton`, `QuestCard`, `XPBar`, `BadgeShowcase`, `StreakDisplay`, `WeeklyRecap`, `CompletionModal`, `SmartSuggestions`. All generic-named (no tavern vocabulary yet).
- **AI Gen**: [`/api/generate/route.ts`](quest-board/src/app/api/generate/route.ts) takes `{location, topic, questType}`, returns a validated quest. Currently isolated on its own page.

## Workflow Diagnosis (the loop today)

| Step | Today | Friction |
|---|---|---|
| **Collect** | `/quests` board with filters; `/generate` separate page | AI generation isolated; "Today" widget exists but doesn't dominate |
| **Track** | Active quests buried inside `/profile` | No dedicated tracking surface; can't see at-a-glance "what am I doing right now" |
| **Complete** | `CompletionModal` fires on quest detail page | Celebration is shallow; no streak push, no shareable artifact |
| **Improve profile** | `/profile` is a 428-line single-page dump | Cluttered; no public version; nothing to share externally |

---

## Design Pillars

1. **Keep the 8-bit DNA, add tavern warmth** — preserve "Press Start 2P", scanlines, pixel borders. *Add* a warm accent layer (candlelight gold `#e8b864`, ember `#c44a36`, oak `#8b5a2b`, parchment `#e8d4a0`) used sparingly as room ambience, not a full palette swap.
2. **The Tavern is the metaphor, not a costume** — every page IS a room with a clear purpose, not just renamed pages. Navigation reads as a place: *Enter the Tavern → Walk to the Quest Board → Sit by the Hearth*.
3. **One job per surface** — split the cluttered `/profile` into a focused **Journal** (tracking) and **Hero** (identity/share).
4. **AI gen as a tavern NPC**, not a separate page — a "Quest Giver" character lives on the Quest Board, asks for your topic + location inline.
5. **Shareability is a first-class feature**, not an afterthought — every completion produces a shareable card; every hero has a public URL.

## The Plan — 6 Phases

Each phase is independently shippable. Phases 1–2 are foundations and should ship first; 3–6 can be parallelized or reordered based on appetite.

---

### Phase 1 — Foundation: Visual Tavern Layer

**Goal**: Establish the tavern visual identity without breaking any existing screens.

**Changes**:
- Extend `tailwind.config.ts` with a `tavern` color group (gold, ember, oak, parchment, candle, smoke) — additive, not replacing `retro.*`.
- Update [globals.css](quest-board/src/app/globals.css):
  - Add a subtle warm vignette / candlelight overlay option (opt-in class, doesn't replace scanlines).
  - Add `.tavern-card` (oak border + parchment-inset variant of pixel cards).
  - Add an animated flicker keyframe for "torch" / "candle" accents.
- Add a small set of pixel art SVG accents in `quest-board/public/tavern/` (banner, candle, mug, scroll, anvil, door, hearth) — start with 6–8 inline SVGs, no heavy asset pipeline.
- Update `<title>` and metadata in [layout.tsx](quest-board/src/app/layout.tsx) — "Tarvn — your adventure tavern".

**Verification**: Visit each existing page. Nothing should look broken; new tavern accents are not yet wired in.

### Phase 2 — Reframed Navigation & Landing (The Tavern Door)

**Goal**: When a user lands on `tarvn.xyz`, they should feel they walked into a tavern.

**Changes**:
- **Landing page** [page.tsx](quest-board/src/app/page.tsx) — rebuild as the tavern entrance: an SVG/CSS pixel-art tavern interior background (hearth on left, quest board center, stairs to rooms on right). Each "spot" in the scene is clickable and routes to a section. Include the hero CTA "Enter the Tavern" → `/board`.
- **Navbar** [Navbar.tsx](quest-board/src/components/Navbar.tsx) — restyle as a wooden sign / parchment scroll. Renamed routes (see below). Add a **Hero pill** on the right: avatar + level + XP bar + gold (when authed). Mobile menu = "Open the door".
- **Renamed routes** (keep old paths as redirects to avoid breaking links):
  - `/quests` → `/board` (The Quest Board)
  - `/profile` → split into `/journal` and `/hero/[handle]`
  - `/generate` → folded into `/board` as the Quest Giver inline panel
  - `/questlines` → `/sagas`
  - `/badges` → `/trophies`
  - `/categories` → `/guilds`
  - (`/hall` leaderboard intentionally deferred — see Phase 5 stretch)
- Update `navLinks` in `Navbar.tsx` and add `next.config.mjs` redirects for the old paths.

**Critical files**:
- [src/app/page.tsx](quest-board/src/app/page.tsx)
- [src/components/Navbar.tsx](quest-board/src/components/Navbar.tsx)
- [src/app/layout.tsx](quest-board/src/app/layout.tsx)
- `next.config.mjs` (add redirects)

**Verification**: Click every navbar link, every landing-page hotspot. All routes resolve. Old `/quests`, `/profile`, etc. URLs redirect cleanly.

### Phase 3 — Collect: The Quest Board with the Quest Giver

**Goal**: One screen handles browsing, suggestions, AND AI generation. The user never feels "I have to go to a different page to make a quest".

**Changes**:
- Restructure [/quests/page.tsx](quest-board/src/app/quests/page.tsx) (now `/board`) into 3 vertical zones:
  1. **Quest Giver panel** (top) — pixel-art NPC sprite + speech bubble. Inline form: "Tell me what you're trying to do, and where you are." Reuses existing `/api/generate` endpoint. Generated quest appears in a card the user can **Accept** or **Discard** without leaving the page.
  2. **Today's Adventure** (middle) — preserve existing "Today / Next Best Quest" + `SmartSuggestions` block, restyled as a parchment notice nailed to the board.
  3. **The Full Board** (bottom) — existing filter-able grid, restyled as wooden quest cards pinned with nails. Tabs become wood-burn tabs.
- Move the manual create flow ([/quests/new](quest-board/src/app/quests/new/page.tsx)) into a small "Write your own quest" link/modal in the Quest Giver panel.
- Restyle [QuestCard.tsx](quest-board/src/components/QuestCard.tsx) as a parchment scroll/wooden plaque variant — small visual upgrade only, no API change.

**Critical files**:
- [src/app/quests/page.tsx](quest-board/src/app/quests/page.tsx)
- [src/components/QuestCard.tsx](quest-board/src/components/QuestCard.tsx)
- New: `src/components/QuestGiverPanel.tsx` (wraps existing generate logic)
- Reuse: [src/app/api/generate/route.ts](quest-board/src/app/api/generate/route.ts) (no backend changes)
- Reuse: [src/components/SmartSuggestions.tsx](quest-board/src/components/SmartSuggestions.tsx)

**Verification**: As a logged-in user, generate an AI quest from `/board` without navigating away, accept it, see it appear in active quests. As a guest, browse the board normally.

### Phase 4 — Track: My Journal + Celebrate at the Hearth

**Goal**: Give "in-progress" work a real home. Make completion feel like a tavern celebration.

**Changes**:
- New page `/journal` — extracted from the active-quests + streak + weekly-recap sections of [/profile](quest-board/src/app/profile/page.tsx). Layout: streak ribbon at top, active quests listed as journal entries (with "log progress" / "mark complete" actions inline), weekly recap pinned at bottom. Reuses `StreakDisplay`, `WeeklyRecap`, `getUserQuestProgressMap`.
- Upgrade [CompletionModal.tsx](quest-board/src/components/CompletionModal.tsx) — "Hearth Celebration":
  - Pixel-art hearth backdrop, pixel mugs raising, "+XP" particles
  - Streak counter pulse if streak ticked up
  - Badge unlock animation if any earned
  - **Two new buttons**: "Pin to Hero Page" (default-on) + "Share Triumph" (generates a card, see Phase 6)
- Profile page (`/profile`) becomes a thin redirect to `/journal` for authed users (preserves bookmarks).

**Critical files**:
- New: `src/app/journal/page.tsx` (lifts code from [profile/page.tsx](quest-board/src/app/profile/page.tsx))
- [src/components/CompletionModal.tsx](quest-board/src/components/CompletionModal.tsx)
- [src/lib/quest-progress.ts](quest-board/src/lib/quest-progress.ts) — reuse all data accessors
- Reuse: `StreakDisplay`, `WeeklyRecap`, `XPBar`

**Verification**: Complete a quest end-to-end. The Hearth celebration plays, XP/streak/badges update, the quest moves from `/journal` Active to `/journal` Recent, and the share/pin actions work.

### Phase 5 — Improve & Show Off: The Hero Page

**Goal**: A focused public "character sheet" worth visiting and sharing.

**Changes**:
- New public page `/hero/[handle]` — character-sheet layout:
  - Top: avatar (introduce a small set of pixel-art portraits — choose from 6–10 sprites), display name, level, XP bar, title (e.g., "Adept of the Fitness Guild" derived from top completed-quest category)
  - Stats panel: total XP, quests completed, longest streak, sagas finished
  - Trophy wall: existing `BadgeShowcase`, full grid
  - "Pinned Triumphs" carousel: 3–5 highlighted completions chosen by user
  - **Public** when `is_public=true`; private fields (email) always hidden when viewer isn't the owner
- Settings surface `/hero/edit` (or modal): pick handle, pick portrait, toggle `is_public`, manage pinned triumphs
- Schema additions (single new migration `006_hero_sharing.sql`, additive only):
  - `profiles.handle` (unique slug for `/hero/[handle]` URL)
  - `profiles.avatar_sprite` (text reference to one of the bundled portraits)
  - `profiles.is_public` (boolean, default true)
  - `profiles.title` (nullable cached derivation)
  - `pinned_quests(user_id, quest_id, position)` — for Pinned Triumphs
  - RLS: public-read on `profiles` is gated by `is_public`; `pinned_quests` readable when the parent profile is public; writes restricted to owner

**Critical files**:
- New: `src/app/hero/[handle]/page.tsx`
- New: `src/app/hero/edit/page.tsx`
- New: `supabase/migrations/006_hero_sharing.sql`
- New: `src/components/HeroPortrait.tsx`, `src/components/PinnedTriumphs.tsx`
- Update: [src/lib/types.ts](quest-board/src/lib/types.ts), [src/lib/quest-progress.ts](quest-board/src/lib/quest-progress.ts) (add hero accessors)

**Verification**: Create a handle, visit `/hero/your-handle` from a logged-out browser tab — only public fields appear. RLS verified: cannot read another user's private fields via direct table query. Toggle `is_public=false` → page 404s for non-owners.

**Stretch (deferred — not in initial commitment)**: a `/hall` Hall of Legends leaderboard. Skipped per scoping decision; can be added later by aggregating the same `profiles` data.

### Phase 6 — Share Triumph: OG Cards & Outbound Sharing

**Goal**: Every completion can become a visible, shareable artifact.

**Changes**:
- New API route `/api/og/triumph/[questId]?user=<handle>` — uses Next.js `ImageResponse` (built-in, no extra dep) to render a pixel-art "TRIUMPH!" card: hero portrait, quest title, XP earned, badge if any, `tarvn.xyz/hero/<handle>` footer.
- "Share Triumph" button in `CompletionModal` and on each completed quest in `/journal`:
  - Web Share API on mobile (`navigator.share`)
  - Copy link + open Twitter/X intent on desktop
- Open Graph metadata on `/hero/[handle]` and `/quests/[id]` so links unfurl with the rendered card.
- Add a small "Share my Hero" button on `/hero/[handle]` (owner view only).

**Critical files**:
- New: `src/app/api/og/triumph/[questId]/route.tsx` (Next.js OG image route)
- New: `src/app/api/og/hero/[handle]/route.tsx`
- Update: [src/components/CompletionModal.tsx](quest-board/src/components/CompletionModal.tsx)
- Update: `src/app/hero/[handle]/page.tsx` — `generateMetadata` for OG tags

**Verification**: Trigger a share from a real completion. Paste the URL into Twitter/X, Slack, iMessage — the card unfurls. Open in incognito, the public hero page renders.

## Cross-cutting Concerns

- **No data loss / breaking changes**: All renames keep redirects. The data model only *adds* columns/tables; never modifies existing constraints.
- **RLS**: Every new table (`pinned_quests`) gets a row-level policy mirroring the pattern in [supabase/schema.sql](quest-board/supabase/schema.sql). New `is_public` profile field gates the `/hero/[handle]` reads.
- **Accessibility**: Pixel font is hard to read at small sizes — keep body copy at ≥9px and add a "Comfort Mode" toggle (Phase 1 stretch) that swaps long copy to a system serif while keeping headers pixel.
- **Perf**: Leaderboard page can ship with a simple sorted query first; revisit with materialized view only if needed.
- **Brand assets**: Inline SVGs for v1 to avoid asset pipeline. Move to optimized PNG/sprite sheet only if the SVG count exceeds ~20.

9. Run `npm run build && npm run lint` clean.

## Decisions Locked

- **Visual treatment**: Pixel + warm tavern accent palette (candle gold, ember red, oak brown, parchment). Press Start 2P stays. No texture/wood-grain pass in v1.
- **Sharing scope**: Public hero page at `/hero/[handle]` + OG share cards. No internal social feed. No leaderboard in v1 (deferred stretch).
- **Schema**: Additive only — `profiles.handle`, `profiles.avatar_sprite`, `profiles.is_public`, `profiles.title`, plus a `pinned_quests` table. No changes to existing constraints.
- **First milestone**: Phase 1 + Phase 2 (visual layer + tavern landing/navbar) ship together with the `tarvn.xyz` domain cutover.
